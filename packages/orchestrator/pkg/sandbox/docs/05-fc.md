# 05. `fc/` Firecracker 进程抽象

> 路径: `packages/orchestrator/pkg/sandbox/fc/`
> 职责: 把 Firecracker 进程当一个普通 Go 对象使用,管理启动脚本、配置序列、暂停/恢复/快照、metrics FIFO 采集、UFFD/memfd 桥接。

---

## 1. 文件分工

| 文件 | 职责 |
|------|------|
| `process.go` | `Process` 类型:启停 FC 子进程、配置 PUT、resume、pause、snapshot、stop 链 |
| `client.go` | `apiClient`:对 FC REST API 的 OpenAPI 客户端包装(loadSnapshot、resumeVM、setBootSource、setRootfsDrive、buildRateLimiter、installBalloon、startBalloonHinting、memoryInfo、dirtyMemory) |
| `config.go` | `Config`/`RootfsPaths`:kernel/FC 版本与路径解析,`arch` 前缀 fallback |
| `kernel_args.go` | `KernelArgs` map → 字符串 |
| `script_builder.go` | 启动脚本(V1 / V2)模板渲染 |
| `mmds.go` | FC 内部 MMDS 元数据结构 |
| `memory.go` | `MemoryInfo` / `DirtyMemory` / `ExportMemory`:FC → 本地 cache(可经 memfd) |
| `fc_metrics.go` | FIFO 读 metrics 协程 + dirty-page throttle 监控 |
| `fph_gates.go` | LD / FC 版本门控(`FCSupportsFreePageHinting` / `FCSupportsMemfd`) |

## 2. `Process` 生命周期

```go
type Process struct {
    Versions              Config
    cmd                   *exec.Cmd
    config                cfg.BuilderConfig
    firecrackerSocketPath string
    metricsPath           string
    slot                  *network.Slot
    rootfsProvider        rootfs.Provider
    rootfsPath, kernelPath string
    files                 *storage.SandboxFiles
    Exit                  *utils.ErrorOnce
    client                *apiClient
    balloonAccum          atomic.Pointer[BalloonMetricsSnapshot]
}
```

### 2.1 `NewProcess` —— 不启动进程,只准备

```go
func NewProcess(ctx, execCtx, config, slot, files, versions, rootfsProvider, rootfsPaths) (*Process, error)
```

- 用 `StartScriptBuilder.Build(versions, files, rootfsPaths, slot.NamespaceID())` 生成启动脚本。
- 检查 firecracker 二进制 + 内核文件存在。
- `exec.CommandContext(execCtx, "unshare", "-m", "--", "bash", "-c", startScript.Value)` —— 用 unshare -m 进入独立 mount ns(让脚本里的 tmpfs 不影响 host)。
- `SysProcAttr.Setsid = true` 创建新 session,防止 Ctrl+C 串到子进程。
- 不在此启动。

### 2.2 `configure` —— 真正启动 + 等 socket

```go
func (p *Process) configure(ctx, sbxMeta, stdoutExt, stderrExt, cgroupFD) error
```

- 装配 stdout/stderr 包装(`zapio.Writer` 写到沙箱 logger,`fcLogFilter` 过滤 `FlushMetrics` 的 request/response 噪音)。
- 若 `cgroupFD != cgroup.NoCgroupFD`,设 `cmd.SysProcAttr.UseCgroupFD = true`(kernel CLONE_INTO_CGROUP,把子进程原子地放进 cgroup)。
- `Mkfifo(metricsPath, 0o600)` 创建 metrics FIFO(FC 会在收到 `PUT /metrics` 后用写端打开)。
- `cmd.Start()` —— 出错时清 FIFO。
- 启动一个 goroutine `cmd.Wait`,把退出状态写到 `Exit`;若是 SIGKILL/SIGTERM 引起的退出,把 err 视为 nil。
- `socket.Wait(ctx, firecrackerSocketPath)` —— 轮询等待 socket 文件出现(10ms tick)。

### 2.3 `Create` —— 冷启动的全配置

`Process.Create` 串行配置(部分可并发但保持简单):

1. 把 `SandboxCacheRootfsLinkPath` 软链到 `/dev/null`(让 FC 先拿到驱动而不真正挂 rootfs)。
2. `configure(...)`。
3. `startMetricsReader(ctx)`(在 `setMetrics` 之前开 O_RDWR 让 FC 写端不阻塞)。
4. `setMetrics(metricsPath)`。
5. 构造 `KernelArgs`(详下)。
6. `setBootSource(kernelArgs, kernelPath)`。
7. 异步 `rootfsProvider.Path()` 拿到真实 rootfs 路径,**用 `utils.SymlinkForce` 替换**之前的 `/dev/null` 软链。
8. `setRootfsDrive(rootfsPath, ioEngine, rateLimiter)`。
9. `setNetworkInterface(vpeer, tapName, tapMac, rateLimiter)`。
10. `setMachineConfig(vCPU, memoryMB, hugePages)` —— ARM64 关 SMT(`Smt = false`)。
11. `setEntropyDevice()` —— 1KB burst、100ms refill。
12. (可选)`installBalloon(freePageReporting, freePageHinting)`。
13. `startVM` —— 同步 action 触发 FC 真正开始执行。
14. (旁路)`setDriveRateLimit` 已在 startVM 后,但需要的话也会写。

### 2.4 `KernelArgs`

```go
args := KernelArgs{
    "quiet":    "",
    "loglevel": "1",
    "init":     options.InitScriptPath,
    "ip":       ipv4,             // format: $ns_ip::$tap_ip:$mask:hostname:iface:dhcp:[$dns]
    "ipv6.disable":  "0",
    "ipv6.autoconf": "1",
    "panic":    "1",
    "reboot":   "k",
    "pci":      "off",
    "i8042.nokbd": "",
    "i8042.noaux": "",
    "random.trust_cpu": "on",
    "rootflags":"discard",        // 关键:ext4 TRIM 配合 reclaim
}
```

- 默认 `quiet` 减少日志;`KernelLogs` 或 `SystemdToKernelLogs` 会清掉 `quiet` 并把 console 路由到 ttyS0。
- `init` 指向 init 脚本(envd / build init)。

### 2.5 `Resume` —— 从 snapshot 恢复

`Process.Resume(ctx, sbxMeta, uffdSocketPath, snapfile, uffdReady, accessToken, cgroupFD, useMemfd, txRL, driveRL)`:

用 errgroup 并发跑三件事:
1. `configure` —— 启 FC 进程。
2. 等 uffd unix socket 出现(`socket.Wait(uffdSocketPath)`)。
3. 拿 rootfs path,`utils.SymlinkForce` 替换 `/dev/null` 软链。

等完成后:
4. `startMetricsReader` + `setMetrics`。
5. `loadSnapshot(uffdSocket, uffdReady, snapfile, useMemfd)`:
   ```go
   backend := &models.MemoryBackend{
       BackendPath: &uffdSocketPath,
       BackendType: ptr(uffd),
   }
   if useMemfd { backend.UseMemfd = ptr(true) }
   LoadSnapshot{Body: SnapshotLoadParams{
       ResumeVM: false, EnableDiffSnapshots: false,
       MemBackend: backend, SnapshotPath: &snapfilePath,
   }}
   // 等 uffdReady
   ```
6. **总是** `setTxRateLimit` + `setDriveRateLimit`(快照可能持久化旧 limit,要覆盖)。
7. `resumeVM`。
8. `setMmds({SandboxID, TemplateID, AccessTokenHash, LogsCollectorAddress})`。

### 2.6 `Pause` / `CreateSnapshot` / `DrainBalloon`

- `Pause` → `PATCH /vm state=Paused`。
- `CreateSnapshot` → `PUT /snapshot`(全量,FC 会先刷盘)。
- `DrainBalloon` → 触发 FPH 循环:
  ```go
  startBalloonHinting(acknowledgeOnStop=true)
  pollFphDone(describeBalloonHinting)  // 5ms → 50ms 指数退避
  ```
  `FREE_PAGE_HINT_DONE = 1`,`DrainBalloon` 是阻塞到 done。

### 2.7 `Stop` —— SIGTERM → 10s → SIGKILL

```go
func (p *Process) Stop(ctx)
```

- 删 metrics FIFO(无 pid 时也尝试)。
- 若 `Exit` 已 done → 立即返回。
- 把 ctx 替换为 `context.WithoutCancel` 避免父 ctx 取消失败。
- 发 SIGTERM:
  - 监听 10s timer,若进程未退,记当前 status(用 gopsutil),再 SIGKILL。
  - 也监听 `Exit.Done()`,提前 return。
- 处理 `os.ErrProcessDone` 视为已退,不报错。

`getProcessStatus` 返回 `R/S/T/I/Z/W/L/D`,用来在 SIGKILL 之前记一行 log。

## 3. `apiClient` —— FC REST 包装

基于 `firecracker-go-sdk` 生成的 client,用 `firecracker.NewUnixSocketTransport` 直连 `firecrackerSocketPath`。

### 3.1 关键方法

| 方法 | 用途 |
|------|------|
| `loadSnapshot` | 包含 memBackend(uffd + 可选 memfd) |
| `resumeVM` | `PATCH /vm state=Resumed` |
| `pauseVM` | `PATCH /vm state=Paused` |
| `createSnapshot` | `PUT /snapshot` 全量 |
| `setMmds` | `PUT /mmds` 写 metadata |
| `flushMetrics` | `CreateSyncAction(FlushMetrics)` |
| `setMetrics(metricsPath)` | `PUT /metrics` 让 FC 把指标写到 FIFO |
| `setBootSource` | `PUT /boot-source` |
| `setRootfsDrive` | `PUT /drives/{id}` |
| `setNetworkInterface` | `PUT /network-interfaces/{id}` + `PUT /mmds-config`(V2,绑 iface) |
| `setMachineConfig` | `PUT /machine-config` (SMT 在 ARM64 关闭) |
| `setEntropyDevice` | `PUT /entropy` |
| `installBalloon` | `PUT /balloon` (amountMib=0,deflateOnOom=false) |
| `startBalloonHinting` | `PATCH /balloon/hinting`(处理 204 被 go-swagger 当作 200 的怪事) |
| `describeBalloonHinting` | `GET /balloon/hinting` 拿 host_cmd |
| `memoryMapping` | `GET /memory/mappings` → `memory.Mapping` |
| `memoryInfo(blockSize)` | `GET /memory` → `DiffMetadata{Dirty, Empty}` |
| `dirtyMemory(blockSize)` | `GET /memory/dirty` → `DiffMetadata{Dirty}` |

### 3.2 `buildRateLimiter` / `buildTokenBucket`

```go
type TokenBucketConfig struct {
    BucketSize, OneTimeBurst, RefillTimeMs int64  // BucketSize<0 关闭
}
```

- `buildTokenBucket`:BucketSize<0 → nil(关掉);否则构造 `models.TokenBucket`,OneTimeBurst>0 才设。
- `buildRateLimiter`:Ops 和 BW 任一非空就返 limiter;两者都空 → nil。

`setTxRateLimit` / `setDriveRateLimit` 特殊处理:**即使 limiter 为 nil,也发一个空 `RateLimiter{}` PATCH**,用来清掉 snapshot 持久化的旧限速。

## 4. `config.go` —— 路径解析

```go
func (t Config) HostKernelPath(config) string {
    archPath := filepath.Join(config.HostKernelsDir, t.KernelVersion, TargetArch(), SandboxKernelFile)
    if _, err := os.Stat(archPath); err == nil { return archPath }
    if !errors.Is(err, os.ErrNotExist) { return archPath }  // 非 NotExist 错误保留 archPath
    return filepath.Join(config.HostKernelsDir, t.KernelVersion, SandboxKernelFile)  // legacy
}
```

- 优先 `{version}/{arch}/vmlinux.bin`(`amd64` / `arm64`)。
- 回退 `{version}/vmlinux.bin` 老路径。
- 路径用 `utils.TargetArch()`(由 `runtime.GOARCH` 决定)。

## 5. `script_builder.go` —— 启动脚本

两个模板:
- `startScriptV1`:用 deprecated rootfs 路径(向后兼容,TemplateVersion <= 1)。
- `startScriptV2`:用 `SandboxDir/rootfs.ext4` 直链,简洁。

模板做的事:
1. `mount --make-rprivate /`(让后续 tmpfs mount 不传播)。
2. `mount -t tmpfs tmpfs $SandboxDir -o X-mount.mkdir`(把整个 sandbox dir 挂成 tmpfs,所有 FC 写的临时文件不会落 host 盘)。
3. 把 host rootfs / kernel 软链到 sandbox dir。
4. `ip netns exec <ns-id> firecracker --api-sock <socket-path>` —— 关键:**在 sandbox 的 netns 里跑 firecracker**,这样 vpeer 接口天然在 ns 里。

`StartScriptBuilder` 在 `NewProcess` 时构造一次,`Build` 出 `StartScriptResult{Value, RootfsPath, KernelPath}`。

## 8. `memory.go` —— 内存导出

### 8.1 `MemoryInfo(ctx, blockSize)` 和 `DirtyMemory(ctx, blockSize)`

直接走 `apiClient` 的对应方法,无 UFFD 时使用:
- `MemoryInfo` 同时返回 resident(等同 dirty)+ empty bitmap —— `NoopMemory` 用。
- `DirtyMemory` 只返回 dirty bitmap —— `Uffd.DiffMetadata` 用。

### 8.2 `exportMemoryFromFc`

```go
func (p *Process) exportMemoryFromFc(ctx, include *Bitmap, cachePath, blockSize) (*Cache, error)
```

1. `m, _ := p.client.memoryMapping(ctx)` —— FC 视角的 user-virt → host-virt 映射。
2. 对 `include` 的每个 range,反查 host-virt 区间(`m.GetHostVirtRanges`)。
3. `block.NewCacheFromProcessMemory(ctx, blockSize, cachePath, pid, remoteRanges)` —— 用 `unix.ProcessVMReadv` 把 FC 进程内存按 host-virt 直接拉到本地 mmap。

`ProcessVMReadv` 是 Linux 跨进程读取的高效系统调用,不需要 pause FC 即可读 —— 所以理论上 `NoopMemory` 也能用,但 FC custom `FREE_PAGE_HINT_STOP` host_cmd 才是 empty bitmap 的来源。

### 8.3 `ExportMemory` 详细

```go
func (p *Process) ExportMemory(
    ctx, include, cachePath, blockSize,
    memfd *block.Memfd,                // FC 用 memfd-export 给的(可能 nil)
    bgCopy bool,                       // 后台 copy 模式
    originalMemfile block.ReadonlyDevice,  // dedup base
    dedupBestEffort, dedupDirectIO bool,
    dedupBudget block.DedupBudget,
    inputEmpty *roaring.Bitmap,
    metaOut *utils.SetOnce[*header.DiffMetadata],
) (block.DiffSource, error)
```

分支(按 memfd 是否非 nil):
- **有 memfd + 有 originalMemfile** → `NewCacheFromMemfdDeduped`:
  - goroutine: `dedupCompare`(对比 base)→ `dedupDrain`(写出 plan.pageDirty)。
  - 同步:metaOut 用 `meta = {pageDirty, pageEmpty+inputEmpty×ratio, BlockSize: PageSize}` 立即设值。
  - 等待 drain 完成后 `done` future 才 resolve,reader 才能访问。
- **有 memfd + 无 base** → `NewCacheFromMemfd` 或 `NewCacheFromMemfdAsync`:
  - metaOut 用 `inputMeta = {include, inputEmpty, blockSize}` 同步设值。
- **无 memfd** → 走 `exportMemoryFromFc`(读 FC 进程内存)。
  - 若有 `originalMemfile` → 再做 `cache.Dedup` 拿到 `dedupCache` + dedup meta。
  - 注意:dedup 后 `blockSize%meta.BlockSize != 0` 时报错。
  - metaOut 用 dedup meta 设值。

`metaOut` 在 defer 里处理错误:任何 sync 失败都 `SetError`,保证 Wait 不会永久挂起。

## 9. `fc_metrics.go` —— Metrics 采集

### 9.1 两 goroutine

`Process.startMetricsReader` 启动两个 goroutine:

1. **Flusher**:每 5s 调 `client.flushMetrics` —— FC 默认 60s 才 flush,主动 flush 让 reader 5s 一帧。
2. **Reader**:
   - `O_RDWR` 打开 metrics FIFO(立即返回,不死等写端)。
   - `O_RDONLY` 打开(因为 O_RDWR 已经存在两侧)。
   - `bufio.Scanner` 逐行读,1MB buffer。
   - JSON 反序列化为 `firecrackerMetrics{Net, Block, Balloon}`。
   - 把所有计数按 TX/RX/Read/Write 维度打到 OTEL 直方图。
   - `balloon` 计数是 SharedIncMetric,每次 flush 是 delta,需要 `accumulateBalloon` 累加到 `balloonAccum`(`atomic.Pointer`)。

Reader 在 `process.Exit.Done()` 时关 O_RDWR,让 O_RDONLY 收到 EOF,scanner 自然退出。

### 9.2 `FlushAndReadBalloonMetrics`

- 调 `flushMetrics`。
- poll `balloonAccum` 变化,2s 超时。
- 返回最新 `BalloonMetricsSnapshot`(给 FPH benchmark 用)。

### 9.3 dirty-page throttle 监控

```go
go monitorDirtyPageThrottle()  // init() 启动
```

每 1s 扫 `/proc/self/task/*/wchan`,数停在 `balance_dirty_pages` 的线程数,加到 `OrchestratorHostBalanceDirtyPagesThreads` 计数器。

`rate()` 趋势反映 GCS 写入触发的内核 dirty-page throttle 强度,0 = 没事。

### 9.4 指标总览

- `orchestrator.sandbox.fc.net.*`:TX/RX 字节、包、丢、限速计数。
- `orchestrator.sandbox.fc.block.*`:read/write 字节、IOEngine 限速、块限速。
- `orchestrator.sandbox.fc.balloon` 通过 `BalloonMetricsSnapshot` 暴露。
- `orchestrator.host.balance_dirty_pages.threads`:全局计数器。

## 10. `fph_gates.go`

```go
func FCSupportsFreePageHinting(fcVersion string) bool
func FCSupportsMemfd(fcVersion string) bool
```

两者都用 `fcversion.New(fcVersion).Has...()`,从 shared `fcversion` 包读版本范围信息。配合 LD flag 双重门控:
- FC 版本太低 → 函数返 false → 不尝试设 balloon/memfd。
- LD flag 关闭 → 也不发 PATCH / PUT memfd。

## 11. 与其他模块的关系

```
sandbox.go
  ├─ CreateSandbox
  │    └─ fc.NewProcess → fc.Process.Create
  │         ├─ configure (unshare -m + setsid + cgroupFD)
  │         ├─ setMetrics (FIFO 写入路径)
  │         ├─ setBootSource / setRootfsDrive / setNetworkInterface
  │         └─ startVM
  └─ Pause
       ├─ bestEffortReclaim
       ├─ process.DrainBalloon
       ├─ process.Pause
       ├─ process.CreateSnapshot
       └─ process.ExportMemory (走 NoopMemory 时)
```

- `fc.Process` 的 `Slot` 字段引用 `network.Slot`,提供 Tap 设备、MAC、namespace、TX rate limit 等。
- `fc.Process` 的 `rootfsProvider` 字段引用 `rootfs.Provider`(`NBDProvider` 或 `DirectProvider`)。
- FC 的 `Exit`(`utils.ErrorOnce`)与 sandbox 的 `exit` 是两个独立对象 —— FC 退出时只 set FC Exit,触发 sandbox 自己的清理协程,再 set sandbox.exit。
